How It Works

From ZIP code input to structured dealer data in four clean, auditable steps. Every layer is independently swappable.

01
Input
ZIP + Config
Submit ZIPs, radius, and adapter selection via the API or admin UI. Bulk import supports CSV upload.
02
Queue
Celery Task
Job enqueued in Redis. Workers pick it up immediately, running each adapter per ZIP with polite crawl delays.
03
Parse
Adapt + Normalize
Site adapter fetches pages, extracts listings, normalizes price, mileage, VIN, and dealer fields into a unified schema.
04
Output
Store + Export
Results upserted to Postgres. Browse via admin UI, filter by make / ZIP, or stream as CSV / JSON export.
